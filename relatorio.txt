O que é Gerenciamento de Configuração de Software (GCS)?
O Gerenciamento de Configuração de Software é a disciplina que visa controlar a evolução de um sistema de software. 
Ele envolve práticas para rastrear e controlar as mudanças no código, na documentação e em outros artefatos do projeto, 
garantindo a integridade e a rastreabilidade ao longo do ciclo de vida do desenvolvimento.

O que é Git?
Git é um sistema de controle de versão distribuído, uma ferramenta fundamental para aplicar o GCS. 
Ele permite que múltiplos desenvolvedores trabalhem em paralelo, registrando um histórico completo de todas as alterações. 
Com o Git, é possível criar ramificações (branches) para desenvolver novas funcionalidades de forma isolada, 
comparar versões e integrar o trabalho da equipe de maneira organizada.

Esta linha foi adicionada diretamente pela interface do GitHub para simular colaboração.

## Reflexão sobre a Experiência

Esta atividade prática foi muito importante para entender o fluxo de trabalho com Git. 
Criar uma branch para uma nova tarefa, fazer commits e depois integrar tudo na branch principal (main) 
simula o dia a dia de um projeto real, mostrando como o Gerenciamento de Configuração é essencial para manter o 
código organizado e o trabalho em equipe eficiente.
